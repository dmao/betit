CSS

As asked, I have not used a css framework. Otherwise, my choice would have been Unuit.css.
But some parts of the code are snippets, like the normalize.css or the code to hide element but sill accessible by screen readers from HTML5 boilerplate
I've decided to use Sass to split the css files into module, with small task. My main.scss call all these files.
I like using prefix to each file to specify the type element we are styling. This usage comes from the Atomic Design by Brad Frost, but I use the same notation than inuit.css.

For the purpose of this test, I don't use sass variable or mixins, the goal wasn't doing a new framework, but nevertheless coding css as if it was a big project.

BEM

Block Element Modifier is the notation that I've used for my classes.

OOCSS

I like also add class to an element, to follow the principle of OOCSS, define by Nicole Sullivan.
It’s an approach for writing CSS that’s fast, maintainable, and standards-based.
Inuit.css use these two approach for his framework: inuitcss is a powerful, Sass-based, BEM, OOCSS framework designed with scalability and performance in mind.
Performance is the main goal when I'm writing CSS, because I try to give the best experience to the customer.

Images

A sprite was given, so I use this image file, but if I have the .psd of the design, I will use two different files, one to serve the regular screen, the other to serve the HDPI screens (Retina).
For the icons, a best solution is to create a font, with a tool like icomoon, so these icon will be retina ready.

I use multiple backgrounds on html element, and use most of the time the before, after properties to add image to elements.
When is not possible, on the input element by example, I use the label element to put the correct image for the field validation.

Images are optimized with a grunt task when I deploy the code.

We can also put the image in the css files, like the logo with the Data URIs technology (but serve regular image for smarphone).

Tools

Grunt

I like working with Grunt, to run some tasks automatically.
My Sass file are watched, my Javascript files are checked by JSHint, the images are optimized, etc.
Once you build the static version of the website, you have versioning of the files (i.e. the name of the file is like main.b4d4abb3.css), they are concatenated and minified.

Chosen

All the form use regular html element dedicated to this task. So the mobil version can use the native form elements.
The buttons for the choice of the sex are radio button for example, but styling the select element is not easy.
So I decide to use a jQuery plugin, to give the same look to each browser and adding image inside.
The Chosen plugin keep the regular select, but use a html dropdown to select an element. You still can reach an element by typing his name, so it's a good UX I think.

I have attached this plugin to the select element only at a specific width, with matchMedia

I keep the sprites given by this plugin to show the usage of media queries to call specific file for Retina screens.

My usage of jQuery with Angular.js is not the best, I should have used directive I think

Assets

With the optimisation in mind, all the assets (images, videos, css, js) must be on CDN so the browser can download multiple items in the same time. Using a CDN has multiple advantages.

With Grunt, the js files are concatenated and minify. Like the css files.
This way, the number of http request are reduced, it's a method to have a faster website, because the number of files than the browser can download in the same time is low. Another example of benefit of using CDN.

I use a file configuration for the server (for Apache here). In this cas, I have the .htaccess made for the html5 boilerplate.
With this configuration, the assets are served gzipped, with expire header in the future, so there are cached.

For example, my CSS file is only 5.1KB and for the second load, this file is not downloaded because the browser use the cached version.

I've seen lectures from Steve Souders and read his books, so I try to use his work to have the fasted website.
Thanks to his job at Yahoo and Google, we have two great tools to do the fastest website: Yslow and PageSpeed.

Testing

On my computer, I have Firefox, Chrome, Safari, Opera, iOS simulator. For IE, I use Virtual Box with different image to have different IE. But In this cas, I've use Browserstack with also Android simulator.
The development version of Chrome (Canary) has also a great tool to emulate device and network.

Conclusions

I like working with performance in mind.
Living in West Africa show me the importance to have fast website. We cannot have a fast network and a slow website don't convert user. They prefer to leave the website to have better experience somewhere else.
Even Google use the speed for the ranking of his result

I don't do website for my ego, but to give the best experience to the visitors of the website.
Using the good html is important for this task, even if this choice implies more works.

Have only a jpg instead of a .psd file is not easy. You can't have the font-family, size, color.
Zooming inside the design to see the details is not possible.
So working in collaboration with the designer is important, because there is always some questins you have to ask.

  • The rays background must be fixed?
  • What is the look when an input text is focused?
  • Animation for button hover?
  • etc

when you do the front end, you are between the design and the developers. You have to talk with these people
